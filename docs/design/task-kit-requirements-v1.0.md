# Task-Kit 要件定義 v1.0(確定)

## 1. 目的
- VS Code workspace に配置して利用する、タスク遂行用エージェントとプロンプト/スキルを提供する。

## 2. 対象範囲

### 2.1 対象範囲(含む)
- CLI で配布物を対象ディレクトリに展開する機能。
- タスクの作成、修正、アーカイブ機能。
- プランの作成、修正、実行機能。
- レビュー用プロンプト生成機能。
- レビュープロンプトを用いたレビュー実行支援機能。
- 報告書の作成、修正機能。
- 課題管理機能(課題の追加、更新、状態管理)。
- 気づきとタスクログの取得機能。
- タスクとプランのリスク検出機能。
- タスク遂行時およびプラン実行時に、AI が事前に推測したリスクとブロック課題を提示する機能。
- タスク引継ぎ資料の作成機能。
- Slack/Teams 向けリード文の作成機能(テンプレート化を含む)。
- 上記機能を実行するためのエージェント定義、プロンプト、スキル、テンプレート群の提供。
- ユーザーが実行中/実行後の成果物で発見した課題を追加し、AI が課題解決のコンサルティングを行う機能。

### 2.2 対象範囲(含まない)
- Slack/Teams への自動投稿そのもの(投稿文生成までを対象とする)。
- 外部SaaS(課題管理ツール、ナレッジ基盤等)との双方向同期。
- モデル学習、ファインチューニング、推論基盤の新規開発。
- 成果物の最終承認を完全自動化する運用。
- ワークスペース外のシステム運用手順の策定。

## 3. 成功条件
- CLI 実行のみで配布物を任意の対象ディレクトリへ展開できる。
- 以下の主要ユースケースが手順化され、再現実行できる。
  - タスク作成/修正
  - プラン作成/修正/実行
  - レビュープロンプト生成とレビュー実行
  - 報告書作成/修正
  - 課題管理
  - 気づき/ログ取得
  - リスク検出
  - 引継ぎ資料作成
  - タスクアーカイブ
  - Slack/Teams 向けリード文作成(テンプレート利用含む)
- 課題管理では、ユーザーが発見した課題を追加でき、AI が解決方針の助言を返せる。
- リスク検出では、プラン実行前または実行中に、想定リスクとブロック課題を提示できる。
- 禁止事項に反する出力(タスクキット利用の明示、記述、考察)が生成されない。
- 各機能の入力テンプレートと受け入れ条件が文書化され、未確定事項が分離されている。
- 現時点では定量指標(成功率、時間)を必須とせず、定性的な受け入れ確認を許容する。

## 4. 制約

### 4.1 暫定制約(提案)
- 既存ディレクトリ構成と既存テンプレートを優先し、無関係な再編をしない。
- 実行環境は Windows または macOS 上の VS Code workspace 利用を前提とする。
- 変更は最小単位で行い、要件、仕様、テンプレートの整合を同一変更で保つ。
- 機密情報、資格情報、個人情報を成果物やログに出力しない。
- インターネット接続を必須とし、オフライン運用は対象外とする。
- 初期導入時は GitHub Copilot の有料プラン加入を必須とする。
- GitHub Copilot 無料プランでの動作は対象外とする。
- メイン利用は GitHub Copilot とし、開発時の補助として Codex を利用可能とする。
- VS Code は 1.102 以上を必須とし、最新版利用を推奨する。
- 実運用で想定するモデルは Claude Sonnet 4.6 以上、および Codex 5.5 以降とする。
- 上記以外のモデルで運用する場合は自己責任とする。

### 4.2 要判断の制約(未確定)
- なし(現時点の制約論点は確定済み)。

## 4.3 CLI エラー受け入れ基準(提案)
- 入力不足エラー: 必須引数または必須設定が不足する場合、終了コードを非0で返し、不足項目を明示する。
- パス不正エラー: 展開先パスが存在しない/書き込み不可の場合、原因と修正例を表示して処理を中断する。
- 権限エラー: 権限不足でファイル作成や更新に失敗した場合、対象パスと必要権限を明示する。
- ネットワークエラー: 初期導入時または実行時に外部接続不能の場合、再試行可否と確認項目(回線、認証状態)を表示する。
- プラン制約エラー: GitHub Copilot 無料プラン等の非対応条件では、非対応理由と必要条件(有料プラン)を明示する。
- 競合エラー: 既存ファイルとの競合時は上書き可否を明示し、意図しない破壊的更新を避ける。
- 予期しない例外: スタックトレースをそのまま露出せず、要約メッセージと調査に必要な最小ログ参照先を提示する。

## 4.4 CLI 終了コード体系(確定)
| 終了コード | 区分 | 意味 | 利用者アクション |
|---|---|---|---|
| 0 | 正常 | 正常完了 | なし |
| 1 | 入力 | 必須引数/設定不足 | 不足項目を指定して再実行 |
| 2 | パス/権限 | パス不正または権限不足 | パスと権限を修正して再実行 |
| 3 | ネットワーク | 外部接続不可/タイムアウト | 回線・プロキシ・認証状態を確認 |
| 4 | プラン制約 | 無料プラン等の非対応条件 | 有料プラン条件を満たして再実行 |
| 5 | 競合 | 既存ファイル競合 | 上書き方針を指定して再実行 |
| 6 | 内部例外 | 予期しない例外 | ログ参照先を確認し再試行/報告 |
| 7 | キャンセル | ユーザーまたは外部中断 | 中断要因を解消して再実行 |

## 5. 禁止事項
- タスクキットで作成した成果物(報告書を含む)に、タスクキットを利用していることを明示、記述、考察として混入しない。
- 秘密情報、資格情報、個人情報を成果物またはログに含めない。

## 6. 決定事項
- 目的は「VS Code workspace で利用可能なタスク遂行支援キットの提供」とする。
- 成功条件は、指定された一連の業務フローを CLI とキット提供物で遂行可能であることとする。
- 禁止事項として「タスクキット利用事実の成果物への混入禁止」を最優先で適用する。
- 対応OSは Windows と macOS とする。
- VS Code は 1.102 以上を必須とし、最新版利用を推奨する。
- 利用モデルは固定しないが、GitHub Copilot 無料プランは対象外とする。
- 動作確認は GitHub Copilot を主軸に実施し、開発時に Codex も利用する。
- オフライン運用は対象外とし、初期導入時のインターネット接続と有料プラン加入を必須とする。
- 定量指標は現時点で必須化せず、定性的な受け入れ確認を採用する。
- 実運用想定モデルは Claude Sonnet 4.6 以上、および Codex 5.5 以降とする。
- 想定外モデル利用時の結果保証は行わず、自己責任で運用する。
- CLI の終了コード体系を 0-7 で固定する。
- タスクアーカイブの保存期間は無期限とし、保持継続可否はユーザー判断とする。
- タスクアーカイブの検索機能は提供要件に含めない。
- 課題/リスク管理テンプレートは最小版を標準として採用する。

## 7. 未解決事項
- なし(本版で解消済み)。

## 8. 判断理由
- ユーザー入力を最優先し、未提示項目のみを推測可能な最小範囲で補完した。
- 未確定の制約を確定事項に混在させると後工程の手戻りが増えるため、明示的に分離した。
- 成功条件は運用で検証できる表現へ寄せ、実行確認可能な粒度に分解した。

## 9. 受け入れ条件
- 目的、対象範囲、成功条件、制約、禁止事項が文書内で区分されている。
- 確定事項と未解決事項が混在せず、追跡可能である。
- 成功条件が実行確認可能な記述になっている。
- 禁止事項が成果物生成フロー全体に適用される形で定義されている。
- CLI の主要エラーパターンについて、利用者が次アクションを判断できるメッセージが定義されている。

## 10. レビュー指摘(自己レビュー)

- 指摘2: 課題管理データ項目は要件本文に方向性を追加したが、最小スキーマはテンプレート整備が必要。
- 指摘3: 想定外モデルの自己責任方針は要件書で確定済み。運用プロンプトへの注意喚起は不要とする。
