# /task-kit.plan-update

目的: 実行計画を更新する。

## 入力
- タスクパス(任意。未指定時は `.task-kit/current-task.md` を参照)
- 計画更新内容(必須)

## 添付テンプレート
- #.task-kit/current-task.md
- #.task-kit/templates/tasks/plan.md

## 手順
1. 添付テンプレート(#.task-kit/current-task.md, #.task-kit/templates/tasks/plan.md)を参照し、タスクパスを確定する。current task が空、不存在、または `tasks/` 配下のタスク領域外なら停止して確認する。未指定時は `.task-kit/current-task.md` のタスクパスを使う。
2. 指定されたタスクパスと `.task-kit/current-task.md` のタスクパスが不一致の場合は、どちらを使うか確認してから進める。
3. plan.md の存在を確認する。不足時は .task-kit/templates/tasks から補完する。
4. 計画更新内容を反映して「4. 実施手順（実施ステップ）」を更新する。
5. 「4. 実施手順」を基に、「3. 作業対象」を更新する。`references` 配下は参照専用として扱い、更新対象に含めない。
6. 「4. 実施手順」を基に、「1. 概要」を AI が要約して更新する。すでに概要がある場合は、更新内容を反映して整合的に再生成する。
7. 「4. 実施手順」を基に、「2. 前提条件」を AI が抽出して更新する。すでに前提条件がある場合は、更新内容を反映して整合的に再生成する。
8. 「4. 実施手順」を基に、「5. レビュー」を AI が類推して更新する。レビューに対する明示指示がある場合はそれを優先する。すでにレビューに記入がある場合は、更新内容を反映して整合的に再生成する。
9. 「6. 備考」はユーザー指示がある場合に追記する。指示がなくても、実施上重要な補足があれば AI 判断で追記してよい。すでに備考がある場合は、更新内容を反映して整合的に再生成する。
10. 実施手順の前提、完了条件、依存関係が曖昧な場合は、推測せず対話で確認して確定する。
11. 実施手順で `references` 配下への出力や更新が指定された場合は実行不可であることをユーザーへ伝え、`outputs` または `records` への代替案を提示する。
12. 実行可能な計画を確定した場合、task.md のステータスと handoff.md の現在地を `計画済み` に更新する。確定した判断は findings.md に、作業中の仮説は scratchpad.md に記録する。

## 出力形式
- 更新対象
- 変更要約
- レビュー更新
- 備考追記(あれば)

## ディレクトリアクセス制約
- タスクディレクトリ配下の `outputs` と `records` は読み書き可とする。`references` は参照のみ可とし、書き込み(作成・更新・削除)を行わない。
- 出力先の明示がない場合は `outputs` を既定の出力先として扱う。

## 禁止事項
- 成果物本文にタスクキット利用事実を混入しない。
