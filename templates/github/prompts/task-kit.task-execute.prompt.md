# /task-kit.task-execute

目的: 承認済み計画を安全に実行する。書込み前に、対象範囲と複雑度に応じたモデル推奨を提示して利用者の確認を得る。

## セッション分離

- 初回の `実行セッションパッケージ` を含まない直接の呼出しでは実行しない。必要な一次入力を参照して参照先中心のパッケージを作成し、新しいセッションを開始して貼り付けるよう案内して終了する。この段階でファイル作成・更新、状態遷移、外部操作は行わない。
- 新しいセッションに貼り付けた `実行セッションパッケージ` は、その対象ステップ群を実行する間のセッション承認として扱う。同一セッション内で、対象タスク・対象範囲・一次入力に実質的な変更がなければ、次の対象ステップへ進むためにパッケージの再掲や新規セッションを要求しない。
- 対象範囲外への拡張、`plan.md` または `issue.md` の実質変更、追加承認、設計変更、モデル変更、コンテキスト不足、独立レビューへの移行が必要な場合は、継続実行せず新しいパッケージまたは利用者確認を求める。パッケージと `plan.md`、`issue.md` が矛盾する場合は推測で実行しない。

## 入力
- タスクパス(任意。未指定時は `.task-kit/current-task.md` を参照)
- 対象ステップ群(任意。未指定時は実行可能な未完了ステップ)
- 実行制約・期待する検証(任意)

## 実行前のモデル推奨
1. `task-kit.execute` は実行セッション開始時、または対象範囲・一次入力が変わる場合に plan.md と issue.md を読み、書込み前に対象ステップ群の複雑度と必要な検証を判定する。
2. 次の層で推奨を提示する。利用可能な具体的なモデル名は呼出し元の選択肢から選び、固定しない。
   - 軽量: 単一ファイルの定型更新、明確な小修正。低コストのモデルを推奨する。
   - 標準: 限定的な実装と通常の検証。通常利用するモデルを推奨する。
   - 高: 複数コンポーネント、非自明な設計判断、セキュリティ・障害影響、未解決依存、広範な検証のいずれかを含む。高性能なモデル、または計画の分割を推奨する。
3. 推奨には、複雑度、根拠、推奨するモデル層、低コスト化できる分割案、実行に必要な検証を含める。
4. 利用者がモデル層または分割案を確認するまで、ファイル作成・更新・状態遷移・外部操作を行わない。同一セッション内で複雑度が変わらない連続ステップには、確認済みのモデル層を継続適用する。別のモデルを選ぶ場合は、そのモデルで本コマンドを再実行する。
5. モデルの推奨・確認自体では実行中のモデルを切り替えられない。選択可能なモデルに推奨層がない場合は、その旨と代替の分割案を示す。

## 実施

利用者確認後の実施、検証、記録、状態遷移は `task-kit.execute` の責務と手順に従う。

## 出力形式
- 結論、変更点（最大5項目）、検証結果、残課題を簡潔に返す。詳細は更新したファイルを参照させる。ただし、ブロッカーと利用者確認事項は省略しない。
