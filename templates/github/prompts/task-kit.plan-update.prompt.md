# /task-kit.plan-update

Task-Kit の計画更新を `task-kit.plan` へ委譲する。

実行可能な計画が確定した場合は、現在の会話で実行を続けず、新しいセッションを開始するよう利用者へ案内する。新セッションへそのまま貼り付けられる、参照先中心の `実行セッションパッケージ` を一つのコードブロックで提示する。コードブロックの先頭行は `## 実行セッションパッケージ`、次行は `/task-kit.task-execute` とし、`タスクパス`、依存関係が明確な連続ステップ群（未指定時は実行可能な未完了ステップ）、`一次入力`（`plan.md`・`issue.md`・必要な補助資料の参照先）、`追加制約`、`検証差分` を含める。追加制約または検証差分がなければ `なし（plan.md に従う）` と記載する。一次入力の本文、会話履歴、未確定事項、秘密情報、他タスクの文脈は含めない。

## 入力
- タスクパス（任意。未指定時は `.task-kit/current-task.md` を参照）
- 計画更新内容（必須）
- 制約・完了条件・レビュー要件（任意）
